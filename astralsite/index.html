Overview

Astral is a tool that helps developers create location-based file-sharing social-media applications. Astral framework allows developers to quickly setup a RESTful API that handles user and file storage and querying.

Close

Technologies

Various technologies were used to create Astral:

Node.JS

From nodejs.org:
Node.js® is a JavaScript runtime built on Chrome's V8 JavaScript engine. Node.js uses an event-driven, non-blocking I/O model that makes it lightweight and efficient. Node.js' package ecosystem, npm, is the largest ecosystem of open source libraries in the world.

MongoDB

MongoDB is a popular, free, and open-source No-SQL database. It's JSON document schema structure made it easy to integrate with a RESTful API.

AWS

RESTful API setup was tested on AWS EC2 Ubuntu instances. Amazon makes it easy for developers to quickly setup an ubuntu server at https://aws.amazon.com/ec2/.

Python and Click

Python (version 3.6) was used to create the astral tool for setting up and installing a complete RESTful API and MongoDB database. Click was used to create a full command-line tool.

Uses

Phylo

Phylo is an application that allows users to share files to specific locations. Phylo is different from other file-sharing applications in that only users at a file's location can view it. This allows users to share digital files in the same way they would share physical ones. Phylo creates a digital world over the existing physical world.

Location-Based File Sharing

In the real world, access to resources is limited by the physical restrictions time and space. In the digital world, files are not restricted in this way. This is typically considered to be one of the greatest benefits of the digital revolution, but there can be instances when such restrictions are desired, if not necessary. Some files should only be accessed at specific locations in the world, at specific times. Astral enforces these restrictions for developers who need them in their applications.

Security

Features

  • Astral allows users to setup their servers with SSL (Secure Socket Layer).
  • Astral never stores user passwords as plaintext. Bcrypt is used to hash passwords before storing.
  • Applications can only be created using the astral command-line tool on the APIs server.
  • JSON Web Tokens are used to authenticate application keys and users.

Limitations

  • No Two-factor authentication. (Can be implemented by applications)
  • Use of passwords is optional. (Can be enforced by applications)
